社交媒体内容创作器 - 使用指南

简介

社交媒体内容创作器是一款浏览器端工具，用于模拟生成各大社交平台的帖子、聊天记录等界面内容。适用于小说创作、AO3 同人文排版、UI 原型设计等场景。

无需安装，直接打开 index.html 即可使用。

快速开始

1. 选择平台

打开页面后，在左侧平台网格中点击目标平台。国内平台和国际平台分为两个标签页。

2. 编辑内容

选择平台后进入编辑器，不同平台提供的编辑选项不同。通用操作包括：

  • 添加消息/评论：点击底部按钮添加新条目
  • 拖拽排序：长按条目左侧的 ⋮⋮ 拖动调整顺序
  • 插入消息：点击条目右上角的 ⬇ 按钮，在其下方插入新消息
  • 切换类型：点击 ⇄ 按钮切换消息方向（发送/收到/通知）
  • 删除条目：点击 ✕ 按钮

3. 预览与导出

编辑时右侧实时预览，完成后使用底部工具栏导出：

  • 导出图片：生成 PNG 图片，自动启用高清模式
  • 导出 HTML：下载完整 HTML 文件
  • 导出兼容 HTML：生成适配 AO3 等平台的精简 HTML（去除动画、SVG 等）
  • 复制 HTML：复制 HTML 片段到剪贴板

各平台详细说明

小红书

  1. 填写作者昵称和头像
  2. 上传帖子图片（支持本地上传和外部链接）
  3. 填写标题和正文，正文中的 #标签 自动高亮
  4. 添加评论：每条评论可设置用户名、头像、内容、时间、点赞数
  5. 评论支持嵌套回复，回复同样支持自定义头像
  6. 底部可设置互动数据（点赞、收藏、评论数）

微信 / QQ 聊天

  1. 设置联系人信息和头像
  2. 发送方消息显示在右侧（蓝色气泡），接收方消息显示在左侧
  3. 支持文本消息、图片消息、文本+图片组合
  4. 支持语音消息（可设置时长和转写文字）
  5. 支持通话记录（语音/视频，可设置通话状态和时长）
  6. QQ 支持在线状态显示和顶部颜色自定义
  7. 微信支持群聊，可设置不同成员的名称和颜色

朋友圈

  1. 添加多条动态，每条动态独立编辑
  2. 每条动态支持最多 9 张图片
  3. 设置点赞列表（逗号分隔昵称）
  4. 添加评论，评论可指定回复对象
  5. 支持定位标签

iMessage

  1. 设置联系人名称、头像、副标题
  2. 发送/收到消息，支持文本和图片
  3. 消息回应（emoji 表情回应）
  4. 通话记录（语音通话 / FaceTime，支持已接听/未接听/已拒绝）
  5. 通知消息：插入系统提示信息（如"该发件人不在你的联系人列表中"），颜色可自定义
  6. 已读回执和正在输入状态

WhatsApp

  1. 单聊或群聊模式
  2. 消息状态标记（单勾/双勾/蓝色双勾）
  3. 语音消息（带播放时长和转写文字）
  4. 主题配色（自定义顶部栏颜色和聊天背景颜色）
  5. 群聊成员颜色自动分配

Instagram / Twitter / Reddit / YouTube

这些社交媒体平台的操作流程相似：

  1. 设置用户信息（头像、昵称/用户名、认证标识）
  2. 编辑帖子/推文内容
  3. 上传图片
  4. 添加评论和回复
  5. 调整互动数据（点赞、转发、收藏等）

平台特色功能：

  • Twitter：话题标签高亮、引用推文、话题串
  • Reddit：Subreddit 显示、Flair 标签（自定义文字和颜色）、投票数、奖项
  • YouTube：视频缩略图、频道信息、置顶评论

综合页面（多平台拼接）

将多个平台的模块组合到一个页面中：

  1. 在单个平台编辑完成后，点击底部「添加到综合页面」按钮
  2. 切换到「综合页面」视图查看所有已添加的模块
  3. 支持拖拽调整模块顺序
  4. 点击模块可返回编辑器修改内容
  5. 支持批量导出：每个模块生成独立的 PNG 图片
  6. 勾选导出：在侧边栏勾选模块后导出兼容 HTML，CSS 包含全部模块样式，HTML 只包含勾选模块，适合 AO3 章节拆分

图片处理

图片上传方式

  • 本地上传：点击上传按钮选择本地图片文件，自动压缩并转为 Base64
  • 外部链接：在 🔗 输入框中填写图片 URL（适用于 AO3 导出，导出时保留链接地址）

两种方式可以同时设置，预览时优先显示本地上传的图片，导出兼容 HTML 时使用外部链接。

头像设置

大多数平台支持自定义头像。未设置头像时会自动生成首字母占位头像。

导出说明

导出图片 (PNG)

  • 使用 html2canvas 渲染，2 倍分辨率输出
  • 自动临时切换到高清模式（SVG 图标），导出完成后恢复
  • 聊天类平台会自动展开所有消息再截图
  • 综合页面导出时，每个模块生成单独的 PNG

导出 HTML

完整的独立 HTML 文件，包含所有 CSS 样式。可直接在浏览器中打开。

导出兼容 HTML

专为 AO3 等受限平台设计：

  • 仅保留白名单内的 HTML 标签和属性
  • 移除 CSS 动画、SVG 样式、交互属性
  • Base64 图片替换为 [图片] 占位符
  • 输出分为 HTML 部分和 CSS 部分，分别粘贴到平台编辑器

复制 HTML

将预览区域的 HTML 和 CSS 复制到剪贴板，方便粘贴到其他工具。

项目管理

保存与加载

  • 点击顶部「保存」按钮保存当前项目到浏览器本地存储
  • 支持保存多个项目，每个项目记录平台、内容和时间戳
  • 通过「加载」按钮选择已保存的项目恢复

配置导入/导出

  • 导出配置：将当前内容导出为 JSON 文件
  • 导入配置：从 JSON 文件恢复内容
  • 适合跨设备迁移或备份

自动保存

页面每 2 秒自动保存当前状态，关闭页面前也会保存。重新打开时自动恢复上次的编辑内容。

预览模式

  • 手机模式 / 桌面模式：切换预览区域的显示宽度
  • 高清模式：将界面中的 Emoji 图标替换为 SVG 矢量图标，提升导出图片质量。导出图片时会自动临时启用

常见问题

Q: 导出的图片中图片/头像显示空白？
A: 确保使用本地上传的图片（Base64），外部链接的图片可能因跨域限制无法被 html2canvas 渲染。

Q: 兼容 HTML 导出后粘贴到 AO3 样式不对？
A: 确保将 CSS 部分粘贴到 AO3 的 Work Skin 功能中设置为工作区皮肤，HTML 部分粘贴到章节内容的「HTML 编辑器」中，在作品信息中选中对应的 Work Skin 即可。

Q: 聊天消息太多时预览区域显示不全？
A: 预览区域可以滚动查看。导出图片时会自动展开显示所有消息。

Q: 数据存在哪里？
A: 所有数据保存在浏览器的 localStorage 中，不会上传到任何服务器。清除浏览器数据会丢失保存的项目。

Q: 支持手机端使用吗？
A: 页面支持响应式布局，可以在手机浏览器中使用，但建议在桌面端使用以获得更好的编辑体验。
